# Example: CodeBuddy Refactor

## User request

`帮我重构这个项目。`

## Recommended route

- primary focus: `need-calibration`
- secondary focus: `codebuddy-code`

## Why

- the request is broad
- full refactoring is expensive
- the user may only need a staged refactor plan or one pilot slice

## Sample response

```markdown
这个任务先不要直接全量重构。你选一个：

[1 只要判断]
[2 先做样本]
[3 分阶段重构]
[4 一次做完]

默认建议：`2` 或 `3`
```
