# Platform Quick Commands

## Purpose

This file provides ultra-short prompts that users can copy directly in CodeBuddy, WorkBuddy, iMA Copilot, and generic chat environments.

The goal is not to explain the whole skill, but to let the user trigger the right behavior in one sentence.

## Generic router prompts

- `请先判断这个任务该进 Ask、Plan、Craft 还是 Expert。`
- `请先用最省 token 的方式回答。`
- `请先不要展开，先告诉我下一步。`
- `请先给我可点选的短选项让我确认。`

## Need calibration prompts

- `这个任务先不要一次做完，请先判断我是要结论、样本、分阶段，还是完整交付。`
- `如果没必要全量处理，请默认先做样本。`
- `先帮我判断这个任务值不值得做成完整版本。`

## WorkBuddy prompts

- `这个 WorkBuddy 任务先只读分析，不要改文件。`
- `这个文件夹先看 3 个文件做样本，不要全量处理。`
- `先列出会读哪些文件、会改哪些文件，再决定执不执行。`
- `请先给我文件范围预算：3个文件、10个文件、一个子文件夹、全目录。`

## CodeBuddy prompts

- `这个 CodeBuddy 任务先只读最相关的 1-3 个文件，不要扫全仓。`
- `这个 bug 先只诊断，别直接改代码。`
- `如果需要改代码，请先只改一个点。`
- `如果需要验证，请先做最小验证，不要直接跑全量测试。`
- `这个重构先分阶段，不要一次性全改。`

## iMA Copilot prompts

- `在 iMA 里请先只看最相关的 3 条材料，不要展开全部知识库。`
- `先给结论，再决定要不要展开。`
- `先列材料，不要直接写完整报告。`
- `如果要我选，请给我 4 个以内的短选项。`

## Button-ready prompt request

If the user wants button-like interaction, they can say:

- `请直接给我可点选的短选项。`
- `请用按钮式确认让我选。`
- `请给我 4 个以内的快捷回复选项。`
