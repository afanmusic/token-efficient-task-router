# Expected Output Checklist

- [ ] 简单明确任务是否进入 Craft Mode
- [ ] 报错咨询是否进入 Ask Mode
- [ ] 复杂多步骤任务是否进入 Plan Mode
- [ ] 文件批量操作是否进入 Plan Mode 并等待确认
- [ ] 高风险任务是否要求确认
- [ ] 重复报错是否建议 Expert Mode
- [ ] 模糊需求是否提出澄清问题
- [ ] 是否避免无意义长篇输出
- [ ] 是否没有未经确认修改核心文件
- [ ] 是否体现 token 节省原则
- [ ] 是否适配 WorkBuddy
- [ ] 是否适配 iMA Copilot
- [ ] 模糊任务是否触发 Interactive Routing Prompt
- [ ] 简单明确任务是否跳过不必要确认
- [ ] 文件操作任务是否询问安全范围
- [ ] 知识库任务是否询问检索范围
- [ ] 用户选择高风险组合时是否触发拦截
- [ ] 是否最多只问 3 个路由问题
- [ ] 是否提供默认推荐模式
- [ ] 是否支持快捷编码回复
- [ ] iMA 首轮是否使用理解锁定句
- [ ] iMA 首轮是否控制在轻量输出范围内
- [ ] iMA 是否默认只看 3-5 条相关材料而非整库展开
- [ ] iMA 是否支持 `1继续`、`2展开`、`3换模式`、`4只结论`、`5只列材料`
- [ ] 是否先进行 focus activation 再加载详细规则
- [ ] 是否最多只激活一个主聚焦和一个辅助聚焦
- [ ] 是否避免把所有 Skill 分支同时展开
- [ ] 只问风险时是否避免进入完整执行方案
- [ ] 大任务是否先进行真实需求校准
- [ ] 是否区分结论、样本、分阶段、一次做完这四种交付粒度
- [ ] 是否默认优先渐进式推进而不是直接全量完成
- [ ] 是否只在范围清晰且用户明确要求时才进入 full batch
- [ ] 是否体现“非必要不消耗”的 token 逻辑
- [ ] 是否提供按钮友好型或快捷回复友好的确认选项
- [ ] 按钮/选项是否控制在 2-4 个以内
- [ ] 按钮/选项标签是否足够短，适合点选或复制
- [ ] 平台不支持按钮时是否仍可作为纯文本选项使用
- [ ] CodeBuddy 是否默认限制读取范围而不是扫描全仓
- [ ] CodeBuddy 是否默认限制编辑范围而不是直接大改
- [ ] CodeBuddy 是否默认限制测试范围而不是直接全量测试
- [ ] CodeBuddy 重构任务是否先做校准和分阶段规划
- [ ] WorkBuddy 是否先确认文件范围预算
- [ ] WorkBuddy 批量任务是否默认先做样本批次
- [ ] 是否提供平台超短调用语句
- [ ] 是否说明已安装技能的协同使用方式
