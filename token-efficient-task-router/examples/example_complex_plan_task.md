# Example: Complex Plan Task

## User request

`帮我重构这个项目。`

## Route

- clarity: partial
- complexity: complex
- risk: high
- token budget: Standard
- mode: Interactive Routing Prompt -> Plan Mode

## Why

The request is broad, likely multi-step, and may touch core files.

## Sample response

```markdown
## 当前任务判断

当前任务适合使用 Plan Mode，因为它涉及项目范围调整，可能影响多个文件和目录结构。

## 目标

先确认重构范围、优先级和不可触碰区域，再设计分阶段重构方案。

## 执行计划

1. 第一步：识别项目结构、核心入口和高风险文件。
2. 第二步：提出最小重构路径和分批变更范围。
3. 第三步：待确认后逐步实施并验证。
```
