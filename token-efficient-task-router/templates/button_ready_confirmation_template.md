## 按钮友好型确认

适用于支持按钮、快捷回复、选项卡或可点击短回复的平台。

如果平台不支持按钮，也可以直接把下面的选项当成可回复文本使用。

### 通用样式

```markdown
为了避免误判和无效消耗，你选一个：

[1 只要结论]
[2 先做样本]
[3 分阶段]
[4 一次做完]

默认建议：`2` 或 `3`
```

### 路由确认样式

```markdown
这个任务先选处理方式：

[1 Ask 只诊断]
[2 Plan 先给计划]
[3 Craft 先做最小版]
[4 Expert 专家视角]

默认建议：`2`
```

### 文件安全确认样式

```markdown
这个任务可能涉及文件操作，请确认：

[1 只读分析]
[2 先给计划]
[3 先做副本]
[4 修改清单后执行]

默认建议：`2` 或 `3`
```

### iMA 轻量知识库确认样式

```markdown
这看起来是知识库任务，你选一个范围：

[1 只看 3 条]
[2 只列材料]
[3 分主题整理]
[4 完整报告]

默认建议：`1` 或 `2`
```

### 继续推进样式

```markdown
如果要继续，你可以点：

[1 继续]
[2 展开]
[3 换模式]
[4 只结论]
```
