# Plan Mode

## Purpose

Plan Mode is the execution staging mode for high-impact or multi-step work.

## Use Plan Mode when

- the request is complex
- the task has multiple steps
- important files are involved
- directories may be restructured
- batch rename, move, delete, or creation may happen
- knowledge bases, databases, deployment, permissions, or automation are involved
- the output may require many tokens or many artifacts

## Required behavior

1. restate the goal
2. list execution steps
3. list impact scope
4. list risks
5. list token-control strategy
6. wait for explicit confirmation

## Forbidden before confirmation

- modifying core business files
- overwriting files
- deleting files
- batch moving files
- batch renaming files
- generating many files
- running high-risk commands
- outputting the full long-form final artifact

## Output target

- default depth: Standard
- target length: 500-1500 Chinese characters
- the output is a plan, not the full result
