# File Safety Rules

## Never do these without confirmation

- overwrite core business files
- delete files
- batch rename files
- batch move files
- generate complex directory trees in bulk
- modify config files
- modify databases
- edit automation scripts
- edit knowledge-base indexes
- modify release artifacts

## File tasks that must enter Plan Mode

- organizing a document library
- restructuring a project folder
- batch naming
- building a knowledge-base taxonomy
- migrating files
- auto-classification
- deleting duplicate files
- editing many documents
- packaging and release tasks
- generating automation scripts

## Required pre-change disclosure

Before file changes, state:

- files to read
- files to modify
- files to create
- files that will not be touched
- whether backup is recommended
- whether confirmation is required

## Safer alternatives

When possible, offer:

- read-only analysis
- a change plan
- new copy generation
- explicit target-file lists
