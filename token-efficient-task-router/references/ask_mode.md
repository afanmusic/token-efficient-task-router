# Ask Mode

## Purpose

Ask Mode is the diagnosis, explanation, and clarification mode.

## Use Ask Mode when

- the user asks why something failed
- the user asks whether a plan is feasible
- the user asks what to do next
- the user asks for advice rather than direct execution
- the request is still vague
- the task may touch important files but permission is not explicit

## Allowed actions

- explain the issue
- diagnose likely causes
- estimate impact
- assess risk
- suggest safe next steps
- ask 1-3 key questions
- provide option sets

## Forbidden actions

- modify core files directly
- run high-risk commands
- delete, overwrite, batch move, or batch rename
- expand into a full implementation by default

## Output target

- default depth: Standard
- target length: 300-1000 Chinese characters
- keep the response structured and decision-oriented

## Default sections

- 判断
- 诊断
- 建议
- 需要确认的问题
